# Competition Demo

## Core claim

PROMETHEUS does not merely generate an answer. It creates competing implementations, makes the winner survive an independent attack, repairs what breaks, and exposes the entire promotion lineage through a live Discord nervous system.

## Seven-minute proof sequence

1. **Mission (45s):** show frozen mission, three strategies, acceptance gates, and execution policy.
2. **Forge (60s):** create three real Git worktrees and show distinct commits.
3. **Standard gate (45s):** all viable candidates run the same tests; deterministic scoring selects the lowest-cost leader.
4. **Adversarial reversal (60s):** the apparently best leader fails boolean/NaN boundary attacks.
5. **Repair (45s):** apply a visible repair commit and rerun both gates.
6. **Promotion proof (60s):** verify receipt hash, event-chain history `[fail, pass]`, artifact hashes, and capability genome.
7. **ServerForge (90s):** run the same startup into the new Discord server; show mission control, adversarial findings, ProofGrid, telemetry, and before/after verification.
8. **Close (30s):** reproduce on a clean runner and display the GitHub Actions proof artifact.

## Judge-facing differentiators

- Counterfactual candidates are isolated code lineages, not three paragraphs in one model response.
- The leader can lose after selection; the adversarial gate has real authority.
- Repair is a committed event, not an erased mistake.
- Promotion evidence is machine-verifiable and portable.
- Capability memory references proof rather than self-description.
- The live Discord surface makes the invisible software factory observable to collaborators and judges.

## Demo failure plan

The local P0 proof requires no network. If Discord is unavailable, show the previously captured live snapshot, apply receipt, and GitHub Actions artifact, then run the offline topology plan. Never substitute a fabricated live claim.

## Recursive finale

After the deterministic seven-minute proof, show the full recursive run as the category-defining extension:

1. Freeze `missions/self-build.json` against the PROMETHEUS repository.
2. Display three Codex worktrees generating simultaneously.
3. Watch matching candidate lifecycle events appear in ServerForge.
4. Show passing-only leader selection and the independent adversarial repair.
5. Verify the committed recursive receipt.
6. Open the promoted draft PR and install the generated wheel.

This proves that the demo mechanism is not confined to the calculator fixture: the same candidate-to-proof law can act on the system that implements it.
